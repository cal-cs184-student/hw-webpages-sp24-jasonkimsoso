Overview

Throughout this project, we are introduced to Bezier curves and compute them through the de Casteljau's algorithm that involve linear interpolations from given control points. Expanding from the 2D Bezier curves, we applied the algorithm to Bezier surfaces. However, Bezier surfaces alone had rigid surfaces and not good shading. So to account for this, I implemented area-weighted vertex normals that is used to provided phong shading and made the surface and shading of the object better. After this, I implemented upsampling to create smoother mesh objects, but to do so we need to define edge flips and splits, which involved pointer manipulation since the mesh structure was stored as a halfedge data structure.


Section I: Bezier Curves and Surfaces

Part 1: Bezier Curves with 1D de Casteljau Subdivision

de Casteljau's algorithm is an algorithm that allows us to evaluate Bezier curves by using recursive linear interpolations from given control points. It is paramterized by the scalar t that varies from 0 to 1. By repeatedly doing recusive linear interpolations we will arrive at a single point that will lie on the Bezier curve from the paramter t. To implement the algorithm, I used a for loop that will evaluate one step of the rescursive algorithm by using the formula $p'_i = (1-t)p_i + tp_{i+1}$, where $p_i$ are the control points; the function evaluateStep() will return a list of linearly interpolated points. Below are images of each step of the Bezier curve with 6 control points using the algorithm specified. We can observe how after each step for a fixed t, the algorithm converges to a single point on the Bezier curve. Also there are two images where one shows the control points being moved to create another Bezier curve and one where I change the parameter t, which we can see the converged point move.


Part 2: Bezier Surfaces with Separable 1D de Casteljau

The de Casteljau algorithm for Bezier surfaces extends the curve algorithm by recursively subdividing a grid of control points. It iteratively interpolates points on the surface by linearly combining control points along rows and columns with (u,v) parameters, respectively. I implemented this by creating a evalaute1D() function that takes in a list of control points and returns a final point that is on the Bezier curve, similar to Part 1. Then using the function, I stored the list of final interpolated points returned by the function evalaute1D() with paramter u, and that list of points was used with paramter v to obtain the final point on the Bezier surface. Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-Weighted Vertex Normals

Vertex normals can be used to obtain phong shading, I implemented these area-weighted vertex normals by traversing through the appropriate halfedges to grab the positions of a vertex. I used this to get the area-weighted average of the surrounding triangles by computing the cross product between the three vertices obtained (that forms a triangle). Using the combined area result, I got the unit vector of that result to obtain the approximate unit normal. Below are the results of area-weighted vertex normals. We can see that the surface got smoother and the shading too.

Part 5: Edge Split

Similar to Part 4, I drew a visualization of edge splits as shown in the image below and changed the pointers of the halfedge accordingly. In total, I declared 6 new halfedges, 3 new edges, 2 new faces, and 1 new vertex. The new vertex v is at the midpoint of the original structure between v0 and v2, with the halfedge being h0. The 6 halfedges are the halfedge and twin pair of the three new edges that are created as a result of the split. For the faces, I updated it's associated halfedges appropriately. By doing this way, it saves the hassle of saving old halfedge data and updating many fields for the original halfedges h1, h2, t1, and t2, since I only had to update the next and face pointers to account for the new halfedges and faces. Although drawing out the pointers did minimize the chances of getting severe bugs, I did have a bug where a split would make a face or surface disappear, but it was quickly fixed when I realized that my original faces f0 and f1 had to have their halfedges updated.

Part 6: Loop Subdivision for Mesh Upsampling

To implement a 4-1 loop subdivision, I computed the new position coordinates for all the vertices in the mesh. Then I computed the updated vertex positions associated with edges. After this step, I basically split every edge in the mesh and flipped any edge that connected a old and new vertex. With this, I saved the vertex positions with the new ones. The sequence and formula's to perform and compute the new position coordinates are shown below. This part was difficult to debug since it relies on the edge split and flip functions. I had instances where sometimes a surface would disappear or my loop subdivision would not properly split because I was not properly defining the new edges and original edges.

As shown below, sharp edges and corners get smoother with each loop subdivision. You can reduce this effect by presplitting some edges, which is shown on the right column.

Another interesting observation is the cube. When you do 4-1 loop subdivision on a cube it becomes asymmetric as shown below. However, you can fix this by predefining some edge splits and flips. I split and flip the edges in a way that made it symetrical when we perform further edge splits and flip, which you can see below on the right. The reason the cube becomes asymmetric is because of the orientation of the triangles and how they are assembled to form this cube. The degree of the vertices was not consistent and varied.
